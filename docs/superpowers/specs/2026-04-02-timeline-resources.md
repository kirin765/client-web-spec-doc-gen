## 속도 최적화 팁

### 1. 사전 준비
```bash
# 리팩토링 전 준비
npm install  # 종속성 확인
npm run dev  # dev 서버 시작 (항상 실행)
```

### 2. VSCode 확장 설치
- **Tailwind CSS IntelliSense** — 클래스 자동완성
- **Color Highlight** — 색상 시각화
- **Prettier** — 자동 포맷팅

### 3. 검색 및 바꾸기 활용
```
Ctrl+H (또는 Cmd+H): Find and Replace
정규식 활성화: Alt+R
대소문자 구분 비활성화: Alt+C
전체 단어 일치: Alt+W
```

### 4. 단축키 활용
```
Ctrl+Shift+P: Command Palette
Ctrl+/: 주석 토글
Ctrl+Z: 실행 취소
Ctrl+Y: 재실행
```

### 5. 병렬 작업
- 여러 파일을 동시에 편집 (VSCode 탭)
- 한 페이지를 리팩토링하는 동안 다른 페이지 준비
- 로컬 dev 서버에서 실시간 검증

## 일정 조정 가이드

### 예상보다 빠른 경우
- ✅ Phase를 조기에 완료
- ✅ Phase 간 검증 시간 감소 가능
- 추가 최적화 (예: 컴포넌트 재사용성 향상)

### 예상보다 느린 경우

| 병목 | 해결책 |
|------|--------|
| MyPage가 복잡함 | 여러 섹션으로 나누어 작업 (PR 분할 고려) |
| 색상 대비 문제 | WCAG 색상 검사 도구 활용 |
| 레이아웃 깨짐 | 모바일 반응형 검증 우선순위 낮추기 |

### 일정 압박 시

1. **MVP 경로 우선**
   - Phase 1 집중 (가장 중요)
   - Phase 2, 3는 나중에 진행

2. **임시 조치**
   - `!` 프리픽스로 빠르게 오버라이드 (`!text-primary-600`)
   - 나중에 정리 가능

3. **자동화**
   - 자동 Search & Replace 스크립트 작성
   - 정규식으로 대량 변환
